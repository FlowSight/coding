# Google Coding Interview Rubrics

## ⚠️ Must Discuss Tradeoffs — Or Get Rejected

Google will reject you in DSA rounds if you just code without discussing tradeoffs. You MUST verbalize:
- Why you chose approach A over B (even if B is valid)
- Separate helper function vs inline logic — when and why?
- Pass by value vs pass by reference — pros/cons (copy cost, mutation safety, clarity)
- Using a set vs sorted array — when does one win?
- Recursive vs iterative — stack overflow risk, readability, tail-call optimization
- Mutable state vs returning new values

**Rule:** Before writing ANY code, discuss at least 2 approaches and their tradeoffs. During coding, narrate design micro-decisions out loud.
