深挖简历项目

和 HR 沟通后得知，这次主要卡在系统设计和简历项目两轮，达到不了对方senior hiring standard。

我自己复盘下来，其实刚面完的时候感觉还不错。系统设计这轮，问的关键点基本都覆盖到了，包括完整的 schema、high-level design，也讨论了多台 WebSocket 的路由、容灾等问题。

简历项目这一轮原本是一小时的面试，实际聊了大概一个半小时。我的项目本身比较复杂，也涉及一定的 domain knowledge，面试官对这个领域并不熟，所以最终只深入聊了一个项目。当时聊完的感觉非常不错，但结果还是no positive signal。

事后反思，可能问题在于：
系统设计太过专注于设计的正确性，但面试官可能更侧重于两人沟通交流
一个项目讲得太久，应该控制在 20 分钟以内
简历项目并不是越复杂越好，更重要的是能否抽象、简化，并在短时间内把事情讲清楚

就当一次经验教训，move on 了。

Given two (singly) linked lists, return the node at which the two lists intersect. If the two linked lists have no intersection at all, return null. Note that the linked lists might contain cycles.

Example 1:

Input: listA = [4,1,8,4,5], listB = [5,6,1,8,4,5]
Output: ListNode(8)
Explanation: The two linked lists intersect at node with value 8.
Example 2:

Input: listA = [1,9,1,2,4], listB = [3,2,4]
Output: ListNode(2)
Explanation: The two linked lists intersect at node with value 2.
Example 3:

Input: listA = [2,6,4], listB = [1,5]
Output: null
Explanation: The two linked lists do not intersect, therefore return null.
Note:

If the two linked lists have no intersection at all, return null.
The linked lists must retain their original structure after the function returns.
You may assume there are no cycles anywhere in the entire linked structure.
Your code should preferably run in O(n) time and use only O(1) memory.
