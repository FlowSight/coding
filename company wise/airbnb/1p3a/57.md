Airbnb


Design a system to manage transactions and refunds.

Requirements:

Given a series of transactions, where each transaction consists of an ID, amount, type ('payment' or 'refund'), and timestamp.
You need to process refunds based on given priority rules.
The system should quickly update with incoming new transactions.
You need to design a data structure to efficiently store and process transactions.
Implement a function process_transactions(transactions: List[Transaction]) -> List[Transaction] that returns the processed list of transactions, where Transaction is an object with ID, amount, type, and status.
Test case:

Input:

transactions = [
  {'ID': 1, 'amount': 100, 'type': 'payment', 'timestamp': 1633036800},
  {'ID': 2, 'amount': 50, 'type': 'refund', 'timestamp': 1633040400},
  {'ID': 3, 'amount': 150, 'type': 'payment', 'timestamp': 1633044000},
  {'ID': 4, 'amount': 50, 'type': 'refund', 'timestamp': 1633047600}
]
Output:

[
  {'ID': 1, 'amount': 100, 'type': 'payment', 'status': 'processed'},
  {'ID': 2, 'amount': 50, 'type': 'refund', 'status': 'processed'},
  {'ID': 3, 'amount': 150, 'type': 'payment', 'status': 'processed'},
  {'ID': 4, 'amount': 50, 'type': 'refund', 'status': 'rejected'}
]
