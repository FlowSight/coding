Problem Description Implement a simulation of the board game Connect Four. The game takes place on a vertically suspended grid with 7 columns and 6 rows. Two players take turns dropping tokens into columns, where the pieces fall straight down to occupy the lowest available space.

Objective The goal is to determine the final state of the grid based on a sequence of moves. The game ends when a player connects four of their tokens horizontally, vertically, or diagonally.

Input A list of integers representing the sequence of columns played by the users.

Output A text-based display of the grid corresponding to the game state at the moment the game finishes.

Rules and Constraints

Termination: The game stops immediately if a winning condition is met or if an invalid move is attempted (e.g., playing in a full column).
Partial Execution: If the game stops early due to a win or error, any remaining moves in the input list must be ignored.
Tokens: Player 1 uses token 0 and Player 2 uses token X.
Code Quality: The solution should focus on structure and readability rather than complexity or optimization.
Example Case Input (Column Full Error): [2, 2, 2, 2, 2, 2, 2, 2, 2, 2, 2]

Analysis: The players alternate placing tokens in column 2. By the 6th move, the column is full. The 7th move triggers the error condition, stopping the game.
