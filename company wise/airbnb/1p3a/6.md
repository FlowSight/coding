From a system design perspective, propose an end-to-end solution that covers at least:

1. Core features and constraints
User login/authentication (high-level design only)
Create a group, delete group (optional), join/leave group
Send and receive group messages
Fetch message history
Unread count and read receipts (state whether you support them and how)
Online/offline scenarios: offline delivery and catch-up strategy
2. API design (examples)
Create group, join/leave group
Send message to a group
Fetch group messages (time/cursor-based pagination)
ACK/read receipt endpoint (if supported)

User
Group
GroupMember
Message (group messages)
Inbox / Delivery / Offset (if needed)

Edge/API layer (HTTP/WebSocket/TCP)

Push/delivery (online push, offline compensation)

Messaging queue/stream (e.g., Kafka/Pulsar)
