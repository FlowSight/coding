My hard-earned Airbnb interview went wrong.
The interviewer was a Chinese guy with three years of experience in the industry. The experience was good; he was friendly and responsive. He asked me a familiar dynamic programming question about sharing rooms.
The question was: Given a list of Properties, each Property has an ID, neighborhood, and capacity. We have a target Neighborhood and a GroupSize. Within the neighborhood, find a list of recommended property combinations with IDs that minimize the number of properties in each combination and ensure the total capacity is greater than or equal to the groupSize, and as close to it as possible. Output one such combination. If no such combination exists, output an empty array.
【Input】
Properties = [ Property(1, "downtown", 5), Property(2, "downtown", 3), Property(3, "downtown", 1), Property(4, "uptown", 4), Property(5, "uptown", 2) ]
Example result:
If Neighborhood = "downtown",
GroupSize: 4 -> {1} GroupSize:
10 -> {} GroupSize
: 6 -> {1, 3}
GroupSize: 3 -> {2}
GroupSize: 16 -> {}
It's important to confirm whether less properties take precedence over closer capacity, and to clarify whether the output should be a single optimal combination or all optimal combinations, as the question may change.
The code needs to be run. The IDE has basic auto-completion functionality. The interviewer only provided a main function, so before attempting the problem, you need to implement the Property class, manually write the input and print results, and then start writing the solution. Finally, you need to manually update the test cases. The interviewer prepared approximately four or five test cases.

Assume we have a list of properties, each containing an ID, a neighborhood, and a capacity. We are given a target neighborhood and a required total capacity (groupSize). Identify a combination of properties within the target neighborhood whose total capacity is greater than or equal to the groupSize and is as close as possible. The number of properties selected should be minimized. If no such combination exists, return an empty list.

Input:
Properties: List[Property], each Property contains (id, neighborhood, capacity)
Neighborhood: str, target neighborhood
GroupSize: int, required total capacity
Output:
List of property ids List[int]
Example:
Properties = [ Property(1, "downtown", 5), Property(2, "downtown", 3), Property(3, "downtown", 1), Property(4, "uptown", 4), Property(5, "uptown", 2) ]

- If Neighborhood = "downtown", GroupSize = 4 -> [1]
- If Neighborhood = "downtown", GroupSize = 10 -> []
- If Neighborhood = "downtown", GroupSize = 6 -> [1, 3]
- If Neighborhood = "downtown", GroupSize = 3 -> [2]
- If Neighborhood = "downtown", GroupSize = 16 -> []
Note:
Confirm if fewer properties are prioritized and whether to return one optimal combination or all optimal combinations.
