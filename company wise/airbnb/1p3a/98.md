roblem Description
Design a banking system that supports recording timestamps for adjustments to an account balance. Implement a method to compute the change in account balance over a specified duration, given a start_time and an end_time.

Requirements:
The start_time should be exclusive, while the end_time should be inclusive.
The system should store and process timestamps sequentially.
Input
Records of balance adjustments, including adjustment time and adjustment amount.
A query period, including start_time and end_time.
Output
The account balance change in the specified time period.
Example
Suppose balance adjustment records are as follows:

[  
  (1000, 50),    # Added 50 at timestamp 1000
  (1500, -20),   # Reduced 20 at timestamp 1500
  (2000, 30)     # Added 30 at timestamp 2000
]
For the period start_time = 1000 and end_time = 2000, the output should be 60 (50-20+30).
