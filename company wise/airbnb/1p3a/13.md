
海投，面了一个组。

一开始hm轮，30分钟。他们主要想知道我现在的scope，然后我问了下expectation。她们说她们已经有一个TL，所以想找个不是TL的IC。根绝这个提前说很合理。很多面staff或者solid senior在找TL的，这个说在前面对双方都有利。

然后是四轮VO。
两轮coding，一轮project deep dive，还有一轮system design，面senior岗位。

coding第一题是task schedule那个题。leetcode没有原题。

第二个coding是get(),set(),getSum()这道题。leetcode也没有原题。

system design是design一个cutomer ticket management的系统。没在面经里见过。

project deepdive。。啊，这轮面完心理很憋屈。我说的项目有short term plan和long term plan，一直被质疑为什么要做long term plan。给了些背景，还是表示不理解。然后问traffic是什么级别，我说每天2B的traffic，然后也没下文。我还说我分析了一些metrics，然后提出了一个方向，leadership也同意了。他问这些metrics是原来就有的，还是你新加的。我说一些是原来就有的，我又加了一些。整体感觉他觉得我的项目没有technical challenge。

其实在大厂很多challenge是来自complexity，cross team collaboration,和reliability，并不一定每个项目都有technical challenge。而且很多core service都提供很好的metrics，自己要加新metric的情况其实很少。最后问他你工作中都有哪些technical challenge，他说we have some our specific challenge，也没说具体的。感觉他们的qps也不高，我也能质疑他们不够challenge。

其实很多中厂都是solve problems already solved by others，或者用open source再改善一下。有些feature确实有technical challenge，但不是所有的feature都有technical challenge。跟以前在同一个大厂的HM也面过behavior，他就说讲一个technical challenge或organizational challenge，这个就很容易说了。想让其他组帮你干活比自己解决一个technical challenge要难很多。最后想听听大家的想法，这种只要technical challenge的情况在面试中怎么说。

Notification system is responsible for sending notifications to the users (customers/hosts/employees) through various platforms like SMS, Push, Email and Tweets.

In Airbnb, we use a notification system to send notifications on a daily basis.

There are 2 types of notifications:

Single (Event triggered)

Bulk (Batch process)

Few Airbnb Use Cases/Background:

Send email as soon as a guest books a listing. (Event triggered)

Send promotional content to potential hosts. (Batch Process)

Send lock code as SMS to the airbnb guest on their check in date. (Event triggered)

Design the notification system at Airbnb scale which can send notifications for event triggered and batch scenarios.

Event triggered scenario would be: If some specific event occurs, send some notification

Batch scenario would be: Send some notification to a defined group of users"
