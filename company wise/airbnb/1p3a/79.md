Design the object model for a ConnectN game. It should support the following features: 1. Set the width, height, and the number of connections required to win at the initialization. 2. Allow players to alternately place pieces on the board. 3. Determine if the current player is winning. 4. Indicate which player wins or if the game ends in a draw after the game finishes.

Given input:

5 5 4
0 0
0 1
1 0
1 1
2 0
2 1
3 0
Expected output:

Player 1 wins
Where 5 5 4 denotes a 5x5 board with 4 connections required to win, and 0 0, 0 1 indicate the columns where players alternately place their pieces.
