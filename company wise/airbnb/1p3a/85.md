System design , job scheduler
The coding exercise is similar to LeetCode, but simpler; you don't need to distribute the strings evenly across each line. There's a follow-up, which isn't difficult either, similar to a newspaper column with a given width on the left and right, where you need to arrange strings within it.
The coding problem involves inputting a pair of tasks. The first pair represents the timestamp (in seconds), and the second pair represents the score for each task. Each task takes one second to execute. Upon completion, the task earns the corresponding score. However, a task must be completed before its designated timestamp. Find the maximum score you can obtain.


Design a job scheduler system capable of effectively organizing and executing different types of tasks. Requirements include implementing functionalities such as task addition, deletion, priority setting, and scheduling strategies.


You are given a list of tasks, each represented by a pair: the first element is the task's timestamp (seconds), and the second element is the task's score. Each task must be completed before the corresponding timestamp, and it takes one second to execute a task. Find the maximum score possible.

Input Description:

The first line contains the number of tasks n (1 <= n <= 10^5).
The next n lines each contain a pair of integers, timestamp and score.
Output Description:

A single integer, the maximum score achievable under the conditions.
Example:

Input:

3
1 3
2 5
3 7
Output:

15
You can execute all three tasks at seconds 1, 2, and 3, respectively, to achieve a total score of 15.
