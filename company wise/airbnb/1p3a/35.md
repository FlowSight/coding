Given a caching system, when the value of a cache entry changes, its dependent cache entries need to be updated as well. Design a function to update the cache to ensure dependencies are correctly maintained. Assume the dependencies between cache entries are represented as a directed acyclic graph (DAG). Design an algorithm to determine which cache entries need to be updated and update them accordingly.

Input:

Nodes represent cache entries, and edges represent dependencies.
A cache entry to be updated along with its new value.
Output:

The updated cache state.
Example:

Initial cache state: {A: 1, B: 2, C: 3, D: 4}
Dependencies: A -> B, B -> C
Update cache entry: A with new value 10

Updated cache state: {A: 10, B: 20, C: 30, D: 4} (assuming A's update causes B and C to update proportionally)
Constraints:

1 <= number of nodes <= 10^4
Try to minimize the update complexity.



Given a list of tasks, where each task has a deadline and a profit associated with completing it. You can choose the order in which to complete the tasks, and completing a task before its deadline earns you the corresponding profit. Design an algorithm to find the maximum profit.

Input:

n: Number of tasks
tasks: A list of tuples where each tuple contains (deadline, profit).
Output:

The maximum total profit.
Example:

Input: n = 3
Tasks: [(1, 20), (2, 15), (2, 10)]
Output: 35
Explanation: Choosing tasks (1, 20) and (2, 15) provides the maximum profit.
Constraints:

1 <= number of tasks <= 10^5
0 <= deadline, profit <= 10^4
